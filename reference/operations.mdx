## Overview

The Primary School Management System implements CRUD (Create, Read, Update, Delete) operations through PHP scripts that interact with a MySQL database. All operations use MySQLi extension for database connectivity.

---

## Database Connection

All PHP operation files establish a database connection using the following pattern:

<CodeGroup>
```php Connection Pattern
<?php
$link = mysqli_connect(
  "sdb-57.hosting.stackcp.net",
  "student84-353031351c89", 
  "ua92-studentAc",

// Check connection
```

## Student Operations

<Tabs>
  <Tab title="Add Student">
    ### Add Student Operation
    
    **File:** `AddStudent.php`  
    **Table:** `Student1`  
    **Operation:** INSERT
    
    #### Process Flow
    
    <Steps>
      <Step title="Check Form Submission">
        Verifies if the form was submitted using `isset($_POST['submit'])`
      </Step>
      
      <Step title="Extract Form Data">
        Retrieves all student fields from POST data
      </Step>
      
      <Step title="Execute INSERT Query">
        Inserts the new student record into the database
      </Step>
      
      <Step title="Display Result">
        Shows success or error message
      </Step>
    </Steps>
    
    #### PHP Variables
    
    <ParamField query="$Sname" type="string">
      Student's first name from form input
    </ParamField>
    
    <ParamField query="$Ssurname" type="string">
      Student's last name from form input
    </ParamField>
    
    <ParamField query="$Sbirthday" type="date">
      Student's date of birth from form input
    </ParamField>
    
    <ParamField query="$Parent_ID" type="string">
      Parent ID (foreign key reference)
    </ParamField>
    
    <ParamField query="$Class_ID" type="string">
      Class ID (foreign key reference)
    </ParamField>
    
    #### Database Query
    
    <CodeGroup>
    ```php Add Student
    if (isset($_POST['submit'])) {
        $Sname = $_POST['Sname'];
        $Ssurname = $_POST['Ssurname'];
        $Sbirthday = $_POST['Sbirthday'];
        $Parent_ID = $_POST['Parent_ID'];
        $Class_ID = $_POST['Class_ID'];
    
        $send = "INSERT INTO Student1 (Sname,Ssurname,Sbirthday,Parent_ID,Class_ID) 
                 VALUES ('$Sname','$Ssurname','$Sbirthday','$Parent_ID','$Class_ID')";
        
        if (mysqli_query($link, $send)) {
          echo "New record created successfully";
        } else {
          echo "Error adding record ";
        }
    }
    ```
    
    ```sql SQL Query
    INSERT INTO Student1 (Sname, Ssurname, Sbirthday, Parent_ID, Class_ID) 
    VALUES ('John', 'Doe', '2015-03-15', '123', '5');
    ```
    </CodeGroup>
  </Tab>
  
  <Tab title="View Students">
    ### View Students Operation
    
    **File:** `ViewStudent.php`  
    **Table:** `Student1`  
    **Operation:** SELECT
    
    #### Process Flow
    
    <Steps>
      <Step title="Connect to Database">
        Establishes MySQLi connection
      </Step>
      
      <Step title="Execute SELECT Query">
        Retrieves all student records
      </Step>
      
      <Step title="Fetch Results">
        Iterates through result set using `fetch_assoc()`
      </Step>
      
      <Step title="Display Data">
        Renders student data in an HTML table
      </Step>
    </Steps>
    
    #### Retrieved Columns
    
    - `Sid` - Student ID (primary key)
    - `Sname` - Student first name
    - `Ssurname` - Student last name
    - `Sbirthday` - Student birthday
    - `Parent_ID` - Associated parent ID
    - `Class_ID` - Associated class ID
    
    #### Database Query
    
    <CodeGroup>
    ```php View Students
    $sql = mysqli_query($link, "SELECT Sid,Sname,Ssurname,Sbirthday,Parent_ID,Class_ID FROM Student1");
    
    while ($row = $sql->fetch_assoc()) {
        echo "
        <tr>
            <th>{$row['Sid']}</th>
            <th>{$row['Sname']}</th>
            <th>{$row['Ssurname']}</th>
            <th>{$row['Sbirthday']}</th>
            <th>{$row['Parent_ID']}</th>
            <th>{$row['Class_ID']}</th>
        </tr>";
    }
    ```
    
    ```sql SQL Query
    SELECT Sid, Sname, Ssurname, Sbirthday, Parent_ID, Class_ID 
    FROM Student1;
    ```
    </CodeGroup>
  </Tab>

## Common Operation Patterns

### INSERT Operations

All INSERT operations follow this pattern:

<Steps>
  <Step title="Verify Submission">
    Check if form was submitted using `isset($_POST['submit'])`
  </Step>
  
  <Step title="Extract POST Data">
    Retrieve all form fields from `$_POST` array
  </Step>
  
  <Step title="Build SQL Query">
    Construct INSERT query with extracted values
  </Step>
  
  <Step title="Execute Query">
    Use `mysqli_query($link, $sql)` to execute
  </Step>
  
  <Step title="Return Feedback">
    Echo success or error message
  </Step>
</Steps>

### SELECT Operations

All SELECT operations follow this pattern:

<Steps>
  <Step title="Execute Query">
    Use `mysqli_query($link, $sql)` with SELECT statement
  </Step>
  
  <Step title="Iterate Results">
    Loop through results using `while ($row = $sql->fetch_assoc())`
  </Step>
  
  <Step title="Display Data">
    Echo HTML table rows with data from `$row` array
  </Step>
</Steps>

### UPDATE Operations

All UPDATE operations follow this pattern:

## Security Vulnerabilities

<Warning>
  **Critical Security Issues:**
  
  The current implementation has severe security vulnerabilities that must be addressed:
</Warning>

<AccordionGroup>
  <Accordion title="SQL Injection" icon="shield-exclamation">
    All queries use direct string concatenation without sanitization:
    
    ```php Vulnerable Code
    $sql = "INSERT INTO Student1 (Sname) VALUES ('$Sname')";
    ```
    
    **Solution:** Use prepared statements with parameterized queries:
    
    ```php Secure Code
    $stmt = $link->prepare("INSERT INTO Student1 (Sname) VALUES (?)");
    $stmt->bind_param("s", $Sname);
    $stmt->execute();
    ```
  </Accordion>
  
  <Accordion title="No Input Validation" icon="filter">
    No server-side validation of user inputs. The system trusts all POST data.
    
    **Solution:** Implement input validation:
    
    ```php Input Validation
    if (empty($_POST['Sname']) || strlen($_POST['Sname']) > 100) {
        die("Invalid student name");
    }
    $Sname = filter_var($_POST['Sname'], FILTER_SANITIZE_STRING);
    ```
  </Accordion>
  
  <Accordion title="Exposed Credentials" icon="key">
    Database credentials are hardcoded in every PHP file.
    
    **Solution:** Use environment variables or configuration files:
    
    ```php Secure Config
    $link = mysqli_connect(
        getenv('DB_HOST'),
        getenv('DB_USER'),
        getenv('DB_PASS'),
        getenv('DB_NAME')
    );
    ```
  </Accordion>
  
  <Accordion title="No CSRF Protection" icon="lock">
    Forms lack CSRF tokens, allowing cross-site request forgery attacks.
    
    **Solution:** Implement CSRF tokens in all forms.
  </Accordion>
  
  <Accordion title="Poor Error Handling" icon="circle-exclamation">
    Generic error messages provide no debugging information but could expose database structure.
    
    **Solution:** Log detailed errors server-side, show generic messages to users.
  </Accordion>
</AccordionGroup>

## Best Practices & Recommendations

<CardGroup cols={2}>
  <Card title="Use Prepared Statements" icon="shield-check">
    Replace all string concatenation queries with prepared statements to prevent SQL injection
  </Card>
  
  <Card title="Input Validation" icon="check-double">
    Validate and sanitize all user inputs on the server side
  </Card>
  
  <Card title="Secure Configuration" icon="gear">
    Move database credentials to environment variables or secure config files
  </Card>
  
  <Card title="Error Handling" icon="bug">

  </Card>
  
  <Card title="Transaction Support" icon="rotate">

## MySQLi Functions Reference

<AccordionGroup>
  <Accordion title="mysqli_connect()">
    Establishes a connection to the MySQL database server.
    
    **Parameters:** host, username, password, database  
    **Returns:** MySQLi link identifier or false on failure
  </Accordion>
  
  <Accordion title="mysqli_query()">
    Executes a SQL query on the database.
    
    **Parameters:** connection link, SQL query string  
    **Returns:** Result object for SELECT queries, true/false for other queries
  </Accordion>
  
  <Accordion title="fetch_assoc()">
    Fetches a result row as an associative array.
    
    **Returns:** Associative array with column names as keys  
    **Note:** Field names are case-sensitive
  </Accordion>
  
  <Accordion title="isset()">
    Checks whether a variable is set and not NULL.
    
    **Returns:** true if variable exists and is not NULL, false otherwise  
    **Usage:** Verifying form submission
  </Accordion>
</AccordionGroup>
